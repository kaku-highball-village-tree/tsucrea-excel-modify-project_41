# 仕様: step0006 プロジェクト名 C∪D vertical の合計欄設定

## 背景
以下の `販管費配賦_step0006` 出力 TSV では、

- `配賦販管費`
- `1Cカンパニー販管費`
- `2Cカンパニー販管費`

の各列について、2行目（値行）が `0` のままになっている。

- `損益計算書_販管費配賦_step0006_2025年04月_A∪B_プロジェクト名_C∪D_vertical.tsv`
- `損益計算書_販管費配賦_step0006_2025年05月_A∪B_プロジェクト名_C∪D_vertical.tsv`

## 要求仕様

### 1. 合計欄の算出対象
- 対象は上記 9 ファイル。
- 対象列は以下 6 列。
  - `配賦販管費`
  - `1Cカンパニー販管費`
  - `2Cカンパニー販管費`
  - `3Cカンパニー販管費`
  - `4Cカンパニー販管費`
  - `事業開発カンパニー販管費`

### 2. 合計欄の定義
- 各対象列に対し、**2行目セルを「合計欄」**として扱う。
- 各対象列の合計欄には、当該列の集計値（定義済みの明細行の総和）を設定する。
- これにより、2行目の既定値 `0` を実データの合計値で置き換える。

### 3. 適用タイミング
- `step0006` の `..._A∪B_プロジェクト名_C∪D_vertical.tsv` 生成時点で、出力前に合計欄を反映する。

### 4. 受け入れ条件
- 対象 9 ファイルのそれぞれで、対象 6 列の2行目が固定 `0` ではなく、列合計値になっていること。
- 0 以外の合計が存在する月では、2行目の値が 0 以外になること。
- 合計が 0 の列については、2行目が 0 のままであること。

- step0006 以外の step 出力仕様変更は行わない。
- 上記 9 ファイル以外への仕様拡張は行わない。
